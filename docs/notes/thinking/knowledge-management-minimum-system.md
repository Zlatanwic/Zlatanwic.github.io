# 知识管理的最小系统

很多知识管理方案一上来就很复杂：标签、数据库、双链、模板、自动化，什么都有。问题是，大多数人根本维护不下去。

我更偏向一个最小系统。

## 最小系统只需要三部分

### 输入

能快速记下问题、想法和例子。

### 组织

能把内容放到少量稳定的分类里，比如写作、工具、思考。

### 输出

能把成熟内容发布成文章、文档或项目说明，而不是永远停留在草稿堆里。

## 为什么要从最小系统开始

- 维护成本低
- 更容易形成习惯
- 更容易看出哪些结构真有用

复杂系统不是不能做，而是应该在最小系统已经运转起来之后，再有选择地补上去。
