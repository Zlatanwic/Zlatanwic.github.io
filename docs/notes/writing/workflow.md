# 我的写作工作流

我偏好的写作方式不是一次写完，而是不断把素材推到更清晰的层级。

## 四个阶段

### 1. 捕捉

先记下问题、判断、例子或一句足够有力的话，不要求完整。

### 2. 聚合

把同一主题下的碎片合到一起，判断它更适合成为博客文章还是知识卡片。

### 3. 成稿

如果内容具备完整叙事，就写成博客；如果重点在复用，就整理进知识库。

### 4. 回流

文章发布后，把其中可复用的部分抽出来，回流到知识库，避免每次都从零开始。

## 为什么这个流程有效

- 它不要求灵感出现时就很完整。
- 它允许博客和知识库互相供给素材。
- 它让内容越来越结构化，而不是越写越散。
