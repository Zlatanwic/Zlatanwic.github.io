# 关于

这里是 Rodebiau 的个人站点，一个把个人主页、博客和知识库合并到一起的数字空间。

## 我想让这个站点解决什么问题

很多个人网站只解决展示问题，不解决沉淀问题。它们像一张静态名片，能告诉别人“我是谁”，却不能继续说明“我在想什么”“我做过什么”“哪些内容值得反复回看”。

这个站点的目标刚好相反：

- 让第一次访问的人能迅速认识我。
- 让持续关注的人能看到我的长期写作和项目演进。
- 让未来的我自己也能把这里当作可检索、可迭代的知识库。

## 站点结构

### 个人主页

首页负责回答三个问题：

1. 我是谁。
2. 我最近在关注什么。
3. 从哪里继续读下去最合适。

### 博客

博客用来承载比较完整的表达，比如：

- 项目复盘
- 方法论文章
- 工具链实践
- 对知识管理和长期创作的思考

### 知识库

知识库更强调复用性，而不是发布感。这里会放：

- 工作流
- 维护清单
- 框架化笔记
- 术语解释
- 工具使用约定

## 我偏好的写作方式

- 优先写对未来仍然有价值的内容。
- 少写“新闻型”文本，多写“结构型”文本。
- 博客负责叙事，知识库负责提炼。
- 尽量让每一篇文章都能成为其它内容的入口。

## 如果你也想搭类似的网站

可以直接参考这里的组织方式：

- 首页聚焦身份与入口，不堆砌时间线。
- 博客负责观点输出，不承载所有碎片。
- 知识库负责长期维护，不追求一次写完。

这会比“只做博客”或“只做笔记站”更适合长期使用。
