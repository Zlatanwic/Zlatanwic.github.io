## 为什么这样组织

这不是一份在线简历，它更像一张持续更新的工作台。

- 首页负责让第一次来到这里的人迅速理解我是谁、关注什么、最近在推进什么。
- 博客负责输出阶段性的观点、方法、复盘与长文。
- 知识库负责沉淀可复用的结构化内容，比如工作流、维护清单、思考框架与工具手册。

把这三部分放在同一个站点里，最大的好处是内容不会彼此孤立。博客文章可以回链到知识卡片，知识库可以不断吸收博客里的成熟段落，主页则把这些资产组织成一条清晰的入口路径。

## 现在这里有什么

- [博客首页](/blog/)：看最新文章与写作方向。
- [知识库首页](/notes/)：看主题分区与入口索引。
- [项目页](/projects)：看我在做什么、为什么做。
- [关于页](/about)：看背景、合作方式与站点说明。

## 推荐浏览路径

### 第一次来

先看 [关于页](/about)，再进入 [项目页](/projects)，用最短路径建立对我的整体印象。

### 想看输出

从 [博客](/blog/) 开始，优先阅读关于数字花园、信息架构与写作系统的文章。

### 想复用方法

直接进入 [知识库](/notes/)，按主题检索工作流、维护清单与最小系统。
