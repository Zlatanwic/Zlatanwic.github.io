# 用 VitePress 搭自己的数字花园

很多人第一次用 VitePress，做出来的其实不是“自己的站点”，而是“VitePress 默认站点换了个标题”。它能跑，但没有身份感，也没有内容结构。

我更在意的是另一件事：能不能用最轻的工具，搭出一个既能写博客、又能放知识卡片、同时还能承担个人主页职责的长期空间。

## 为什么是 VitePress

我喜欢它有几个原因：

- Markdown 优先，写作成本低。
- 结构天然清晰，适合知识库。
- 主题定制不算重，足够表达风格。
- 静态生成简单，长期维护成本可控。

## 这个站点真正需要补的不是页面，而是结构

默认示例往往只有一个首页和几篇演示文档。但对个人站来说，真正重要的是回答这几个问题：

1. 新访客第一眼看到什么。
2. 老读者怎么快速找到最新文章。
3. 自己以后怎么回头找到曾经写过的内容。

如果这三个问题没有被设计过，站点就很容易变成一个“内容有了，但没人愿意逛第二次”的地方。

## 我更推荐的三层结构

### 第一层：主页

主页不是简历，也不是文章列表。它是入口，负责快速建立认知。

### 第二层：博客

博客负责承载长文、观点和复盘，是“完整表达”的区域。

### 第三层：知识库

知识库负责沉淀那些应该被反复引用的内容，比如工作流、术语、维护清单、方法模型。

## 最后

如果你也想做一个长期站点，先别急着填很多内容。先把骨架搭对，让未来的内容有地方长。
