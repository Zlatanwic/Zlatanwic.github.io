# 写作说明

博客不是独立存在的内容容器，它和知识库是互相喂养的。

## 一篇文章通常怎么来

1. 先从项目、实践或碎片笔记里冒出一个问题。
2. 如果问题足够完整，就写成博客文章。
3. 如果里面有可复用的结构，再拆回知识库。

## 什么适合写成博客

- 有明显叙事线
- 有完整上下文
- 有一个值得展开的判断或观点

## 什么适合沉淀进知识库

- 可复用的步骤
- 术语定义
- 检查清单
- 方法框架

这套分工能减少“所有内容都堆到博客里”或者“所有笔记都散在角落里”的问题。
